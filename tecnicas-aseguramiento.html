Técnicas de Aseguramiento de Calidad

Existen distintos conceptos que ayudan a comprender las técnicas de aseguramiento de la calidad, entre ellos la verificación y la validación.

  • Verificación: Son actividades que se centran en confirmar la correcta construcción de un producto de software, que se cumplan las especificaciones técnicas y no tenga errores de código, arquitectura o documentación.
  • Validación: Son actividades que ayudan a asegurar si el producto que se está construyendo es el correcto; el cual se asegurará de que cumpla con los requisitos funcionales y no funcionales.

Para poder asegurar la calidad de un software, se utilizan distintas técnicas que puedan garantizar los estándares definidos.

Tipos de Pruebas

Los tipos de pruebas suelen ser:

  • Funcionales: Son pruebas que se encargan de verificar que el sistema funcione correctamente, o sea que cumpla su función. Por ejemplo: que el botón de ‘Login’ permita el acceso solo con los datos correctos.
  • No funcionales: Son pruebas que se enfocan en medir el rendimiento del software, su seguridad y usabilidad. Por ejemplo, que la aplicación soporte determinada cantidad de usuarios conectados al mismo tiempo.
  • Automatizadas: Son pruebas que ejecuta el mismo software, por lo tanto, se ejecutan mediante herramientas o scripts que simulan la interacción humana. Por ejemplo: Selenium, JUnit. Cypress.
  • Manuales: Son pruebas que realizan evaluadores humanos, que verifican el funcionamiento siguiendo casos de pruebas.

Actividades de Verificación de Software

Las actividades de verificación de software incluyen:

  • Unidad: Para componentes individuales, validan el correcto funcionamiento de componentes individuales.
  • Integración: Para módulos combinados, con enfoque en la verificación de la interacción entre módulos o subsistemas.
  • Sistema: Para la aplicación completa, por lo tanto válida el software en su conjunto.
  • Aceptación: Válida la satisfacción del usuario.

Técnicas de Caja Negra y Caja Blanca

Las técnicas que utiliza pueden incluir la caja negra y la caja blanca:

  • Pruebas de caja blanca: Están enfocadas en pruebas de programa, donde las pruebas se basan en el conocimiento de la estructura del programa y sus componentes. El acceso al código fuente es esencial para las pruebas de caja blanca.
  • Pruebas de caja negra: Tienen enfoque en las pruebas donde los examinadores no tienen acceso al código fuente de un sistema o de sus componentes. Las pruebas se derivan de la especificación del sistema.

Aseguramiento de Calidad

El aseguramiento de la Calidad de Software hace referencia al conjunto de procesos y actividades por el que pasa el producto, las cuales forman parte del ciclo de desarrollo que se diseñó para evaluar y mejorar la calidad de productos digitales.

El objetivo principal del aseguramiento de la calidad en el desarrollo de software es poder garantizar que los productos digitales funcionen de forma correcta independientemente del dispositivo o su sistema operativo utilizado por los usuarios, para esto se hace uso de varios procesos y técnicas centradas en mejorar el ciclo de desarrollo de software para poder evitar cualquier falla antes del lanzamiento del software.

Hay buenas técnicas de aseguramiento de calidad adicionales; como por ejemplo la integración continua para detectar errores rápidamente a través de pruebas automáticas frecuentes. También el desarrollo guiado por pruebas (TDD) que va a impulsar a escribir pruebas antes del código. Luego las revisiones de código para asegurar la calidad antes de la integración final.
