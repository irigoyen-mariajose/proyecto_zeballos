RECURSOS Y APRENDIZAJE

Glosario de Términos

Ciclo de vida del software:

Con frecuencia se usa como otro nombre para el proceso de software; originalmente se acuñó para referirse al modelo en cascada del proceso de software.

Scrum

es un método de desarrollo ágil que está basado en sprints: ciclos cortos de desarrollo. Puede utilizarse como una base para la gestión de proyectos ágiles, junto con otros métodos ágiles como XP.

ISO 9000/9001

Conjunto de estándares o normas para procesos de gestión de la calidad definidos por la International Standards Organization (ISO). ISO 9001 es el estándar ISO que resulta más aplicable al desarrollo de software. Puede usarse para certificar los procesos de gestión de calidad en una organización.

Programación Extrema (XP)

Método ágil de desarrollo de software usado ampliamente, que incluye prácticas como requerimientos basados en escenarios, desarrollo de primera prueba y programación en pares.

Validacion

Proceso de comprobar que un sistema satisface las necesidades y expectativas del cliente.

Modelo Cascada

Modelo de proceso de software que comprende etapas de desarrollo discretas: especificación, diseño, implementación, pruebas y mantenimiento. En principio, una etapa debe completarse antes de que sea posible el avance a la siguiente etapa. En la práctica, existe significativa iteración entre etapas.

Liberaciones

conjunto de funcionalidades mínimas utiles para el usuario de valor para el negocio. Son frecuentes y se agregan incrementalmente a la primera liberación sumando funcionalidades.

Preguntas Frecuentes

La gestión de calidad de software es el conjunto de actividades que aseguran que el producto final cumpla con los requisitos y expectativas del cliente. Incluye procesos de planificación, control y mejora continua de la calidad a lo largo de todo el ciclo de vida del desarrollo.

La calidad de software es que tanto un sistema, componente o proceso cumple con los requisitos especificados, cumplir las necesidades del cliente y sus expectativas. Los factores que la determinan son:

  • Confiabilidad que funcione sin fallas.
  • Usabilidad que sea sencilla de usar.
  • Eficiencia: optimizar los recursos del sistema.
  • Mantenibilidadoptimizar los recursos del sistema.
  • Portabilidad:ejecutable en distintos entornos.

Un sprint es un ciclo de trabajo que suele durar un determinado tiempo en el marco de trabajo Scrum. Cuando termina el ciclo, se entrega un incremento del producto, que es una parte del proyecto completamente funcional para el uso de los usuarios.

Para elegir la metodología adecuada considera:

  • Tamaño y complejidad del proyecto: Proyectos grandes pueden beneficiarse de Scrum, mientras proyectos pequeños pueden usar Kanban
  • Experiencia del equipo: Equipos novatos pueden empezar con metodologías más simples
  • Nivel de incertidumbre: Alta incertidumbre favorece metodologías ágiles
  • Requisitos del cliente: Si los requisitos cambian frecuentemente, opta por metodologías flexibles
  • Cultura organizacional: La metodología debe alinearse con la cultura de la empresa

Es un ciclo de trabajo repetitivo que dura un determinado periodo de tiempo en un proceso de desarrollo de software. En cada iteración el equipo debe planificar, desarrollar, poner a prueba y revisar las tareas y objetivos a cumplir del software. Es similar al sprint ya que comparten muchas características, la diferencia está en que la iteración tiene como objetivo entregar mejoras de forma incremental y recibir una retroalimentación, ya que todo sprint puede tener una iteración, pero estas no todas necesariamente son sprints. Las iteraciones pueden adaptarse a distintas metodologías, es más flexible ya que no cuenta con una estructura fija determinante , tampoco no cuenta con ceremonias ni sus entregas deben ser entregables sino que avances.

Ejercicios Prácticos

Planificación de Sprint Fácil
Aprende a planificar un sprint de dos semanas usando la metodología Scrum.
  1. Crea una lista de productos con historias de usuario
  2. Estima cada historia usando puntos de esfuerzo
  3. Selecciona las historias para el sprint de dos semanas
  4. Define el objetivo del sprint
Reunión Diaria de Equipo Fácil
Practica cómo realizar una reunión diaria de seguimiento en Scrum.
  1. Asigna los roles: Maestro Scrum, Dueño de Producto, Equipo de Desarrollo
  2. Prepara las respuestas: ¿Qué hice ayer? ¿Qué haré hoy? ¿Qué problemas tengo?
  3. Realiza la reunión en un máximo de 15 minutos
Revisión de Código Medio
Aprende a revisar código fuente para identificar problemas y mejorar la calidad.
  1. Analiza un código de ejemplo proporcionado
  2. Identifica problemas de legibilidad y rendimiento
  3. Propone mejoras para optimizar el código
Creación de Pruebas Medio
Diseña casos de prueba para una función de inicio de sesión.
  1. Define los requisitos de la función de inicio de sesión
  2. Crea casos de prueba: inicio exitoso, correo inválido, contraseña corta
  3. Usa el formato Dado-Cuando-Entonces para estructurar las pruebas
Mejora de Código Medio
Aprende a mejorar la estructura de código existente aplicando buenas prácticas.
  1. Analiza un código que presenta problemas de diseño
  2. Aplica los principios SOLID para mejorar la estructura
  3. Utiliza patrones de diseño para optimizar la solución
Diseño de Interfaz Web Medio
Diseña una interfaz web para un sistema de gestión de usuarios.
  1. Diseña los puntos finales para crear, leer, actualizar y eliminar usuarios
  2. Define los métodos HTTP y las direcciones URL para cada operación
  3. Incluye sistema de autenticación y control de permisos
